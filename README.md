# FailureOnTheFly

FailureOnTheFly is a web-based training simulator designed to help (future) teachers develop their diagnostic competencies. It allows users to engage in simulated conversations with students to uncover mathematical misconceptions.

## 🚀 Overview

The application simulates a classroom situation where a student has made a specific error in a math problem. The teacher's goal is to:
1. **Interact**: Ask questions via text or voice to understand the student's thinking.
2. **Analyze**: Use the student's responses (generated by an LLM) to identify the underlying error pattern.
3. **Diagnose**: Submit a formal diagnosis choosing from predefined patterns and providing a free-text explanation.

## ✨ Key Features

- **Simulated Student (LLM)**: Realistic interaction based on specific pedagogical error patterns.
- **Voice Interaction**: Integrated speech-to-text (OpenAI Whisper) for natural communication.
- **Structured Scenarios**: Predefined tasks with context, student work, and correct solutions.
- **Diagnosis Workflow**: Formal closing of the diagnostic loop with feedback.
- **SDD Methodology**: Developed using Specify-Driven Development for high architectural integrity.

## 🛠 Tech Stack

- **Frontend**: [Streamlit](https://streamlit.io/)
- **LLM Integration**: [Instructor](https://github.com/jxnl/instructor) (Structured outputs)
- **Providers**: OpenAI (GPT-4o-mini, Whisper-1)
- **Language**: Python 3.11+

## 🚦 Getting Started

### Prerequisites
- Python 3.11+
- OpenAI API Key

### Installation
1. Clone the repository.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Configure your environment:
   ```bash
   cp .env.example .env
   # Add your OPENAI_API_KEY to .env
   ```
4. Run the application:
   ```bash
   streamlit run app.py
   ```
